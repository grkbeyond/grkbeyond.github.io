Handle解析

发表于 2015-05-10 | 分类于 Android

Handler 是android提供的一套用来解决同一个进程的线程间通信的一套消息机制。主要包括Handler和Handler所附带的MessageQueue和Looper的工作过程，这三者是一个整体。

概述

Hander,Message,MessageQueue,Looper 概念

1，Handler：消息辅助类，主要功能向消息池发送各种消息事件
2，Message：消息分为硬件产生的消息(如按钮、触摸)和软件生成的消息。
3，MessageQueue：内部存储一组消息。主要功能是主要功能向消息池投递消息(MessageQueue.enqueueMessage)和取走消息池的消息(MessageQueue.next)。采用的是单链表的数据结构存储消息。
4，Looper：消息循环器，以无限循环(Looper.loop)的方式将消息分发给目标处理者。

Hander,Message,MessageQueue,Looper关系

1，Handler中有一个Looper和一个MessageQueue。在Handler的构造函数中 mLooper = Looper.myLooper();将当前线程唯一的looper赋给Handler的mLooper变量。 mQueue = mLooper.mQueue; 将Looper中的MessageQueue赋值给Handler的mQueue变量。
2，Looper有一个MessageQueue消息队列；在Looper构造函数中会初始化MessageQueue。
3，MessageQueue有一组待处理的Message；
4，Message中有一个用于处理消息的Handler；在将消息放到消息队列之前，会将当前handler赋给Message里的target变量。 msg.target = this。

总结

所以在主线程中可以直接创建Handler对象，而在子线程中需要先调用Looper.prepare()才能创建Handler对象。顺便介绍下上面提到的一些方法和对象

ThreadLocal

线程本地存储区。Android中每个线程都有自己的私有本地存储区域，不同线程之间彼此不能访问对方的ThreadLocal区域。ThreadLocal最主要的两个方法，set()和get()他们的操作类型都是泛型。可以看到在Looper中的sThreadLocal变量，其定义如下：static final ThreadLocal sThreadLocal = new ThreadLocal();可见sThreadLocal的get()和set()操作的类型都是Looper类型。

Looper.prepare()

Looper.prepare()在每个线程只允许执行一次，该方法会创建Looper对象，Looper的构造方法中会创建一个MessageQueue对象，再将Looper对象保存到当前线程ThreadLocal中。

Looper.myLooper()

在Handler创建的时候调用，从当前线程ThreadLocal中得到对应的Looper对象。

发送消息

一般情况我们先创建一个带有数据信息的Message对象，然后通过Handler的sendMessage()方法把消息发送出去，然后在Handler的handleMessage()方法中对信息进行相应处理。其实在Handler中不止一个sendMessage()方法可以用来发送消息，其中除了sendMessageAtFrontOfQueue()方法之外，其它的发送消息方法都会调直接或者间接用sendMessageAtTime()这个方法，关系如下图:

从上面可以看到发送消息之后最后调用的是enqueueMessage()，这是Handler发送消息的最后一步。这里我们先来分析enqueueMessage()这个方法。

在enqueueMessage()中，首先会把当前的Handler对象赋值给Message的this字段。这样Message就持有当前的Handler对象。(这是是为了能在Looper的loop()方法中从消息队列中取出消息后通过Message持有的Handler调用handleMessage()方法处理消息)接着就是消息队列的入队方法。

从上面可以看到MessageQueue是按照Message触发时间的先后顺序排列的。如果你是通过sendMessageAtFrontOfQueue()方法来发送消息的，它也会调用enqueueMessage()来让消息入队，只不过时间为0，从而使Message加入到消息队列的队头。MessageQueue采用的是单链表的数据结构，每一个Message都会有next和pre字段，分别指向后一个Message和前一个Message。至此，一个Message就会被放入到消息队列中。

Looper

经过上面的过程，一个Message就被放入相关联的MessageQueue中，那什么时候取，怎么取出来处理呢？这时候Looper就起作用了，在Android中Looper被当作是一个消息轮询器，它会不停地从MessageQueue中查看是否有新消息，如果有新消息就会立刻处理，否则就一直阻塞在那里。在分析Handler创建的时候。我们已经知道Looper在创建的时候会在构造函数中创建一个MessageQueue。也知道通过Looper.prepare()方法可以在子线程中创建一个Looper。在Looper中最重要的一个方法是loop()方法，在这个方法里，会去取MessageQueue中的消息然后调用msg.target.dispatchMessage(msg)。

当Looper的quit方法被调用时，Looper就会调用MessageQueue的quit或者quitSafely方法来通知消息队列退出，当消息队列被标记为退出状态时，它的next方法就会返回null。这时候loop就会跳出循环。在loop方法中会调用MessageQueue的next()方法取出消息。next()是一个阻塞操作。当没有消息时，next会一直阻塞在那里。如果有消息，next()方法就会返回这条消息，并且调用msg.target.dispatchMessage(msg);这里的msg.target就是发送这条消息的Handler.

Handelr.dispatchMessage()

1,首先。如果Message的回调方法callback不为空时，则调用 message.callback.run()方法。callBack是Runnable的数据类型。实际上就是Handler的post方法所传递的Runnable参数。
2,其次，如果Handler的mCallback成员变量不为空时，则回调方法mCallback.handleMessage(msg)。Callback是一个接口，可以通过Callback创建Handler对象：Handler handler = new Handler(callback)。在日常开发中，我们创建Handler对象的时候一般都是派生一个Handler的子类并重写其handleMessage方法来处理具体的消息。而通过Callback就可以不用派生子类就可以创建Handler对象。
3,最后调用Handler自身的回调方法handleMessage(),该方法默认为空，Handler子类需要复写这个方法实现具体的处理逻辑。至此，至少在java层面，我们已经知道了为什么在handleMessage()方法中可以获取到之前发送的消息。

Handler消息机制流程总结


1，Handler通过sendMessage()发送Message到MessageQueue队列；
2，Looper通过loop()，不断提取出达到触发条件的Message，并将Message交给target来处理；
3，经过dispatchMessage()后，交回给Handler的handleMessage()来进行相应地处理。
